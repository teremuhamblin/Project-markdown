📁 Structure pour :
# >> Project‑markdown

> Voici une structure professionnelle, minimaliste, scalable, parfaitement adaptée à un projet basé sur la documentation Markdown :

```rst
project-markdown/
│
├── README.md
├── PLAN.md
├── STYLEGUIDE.md
├── EXEMPLE.md
├── TEMPLATE.md
├── CHANGELOG.md
├── CONTRIBUTING.md
├── CODEOFCONDUCT.md
├── SECURITY.md
├── LICENSE
│
├── docs/
│   ├── README.md
│   ├── writing/
│   │   ├── README.md
│   │   ├── rules.md
│   │   └── examples.md
│   │
│   ├── project/
│   │   ├── README.md
│   │   ├── structure.md
│   │   └── workflow.md
│   │
│   └── templates/
│       ├── README.md
│       └── document-template.md
│
└── assets/
    ├── README.md
    └── images/
```

---

🧩 Pourquoi cette structure ?

✔️ docs/
>Pour centraliser toute la documentation avancée.

✔️ docs/writing/
>Pour tout ce qui concerne l’écriture, les règles, les exemples.

✔️ docs/project/
>Pour documenter le projet lui‑même : structure, workflow, organisation.

✔️ docs/templates/
>Pour stocker les modèles Markdown (TEMPLATE.md, variantes, etc.).

✔️ assets/
>Pour les images, captures, ressources visuelles.
