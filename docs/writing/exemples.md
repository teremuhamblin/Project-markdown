`markdown

Exemples d’écriture

Ce fichier fournit des exemples concrets pour illustrer les règles d’écriture définies dans rules.md.  
Il sert de référence pratique pour comprendre comment structurer, formuler et présenter les contenus du dossier writing/.

---

1. Exemple : paragraphe clair
> Un paragraphe doit être court, précis et centré sur une seule idée.  
> Il doit permettre au lecteur de comprendre immédiatement l’objectif du texte sans effort.

---

2. Exemple : liste structurée
Une liste doit être utilisée pour organiser des éléments similaires ou successifs.

- Objectif du fichier
- Étapes à suivre
- Résultat attendu
- Notes complémentaires

---

3. Exemple : mise en évidence
- Pour attirer l’attention sur un concept clé : utiliser le gras.
- Pour un terme technique ou un élément de syntaxe : utiliser code inline.
- Pour un bloc de configuration ou un extrait long :

`markdown
{
  "name": "project",
  "version": "1.0.0"
}
`

---

4. Exemple : section bien organisée

1. Contexte
Brève description du sujet, de la problématique ou de l’objectif.

2. Objectif
Ce que le lecteur doit comprendre, apprendre ou accomplir.

3. Instructions
1. Étape 1 : action simple et claire.
2. Étape 2 : explication ou précision.
3. Étape 3 : résultat attendu.

---

5. Exemple : définition d’un terme
Token : élément de base du design system représentant une valeur unique (couleur, taille, espacement).

---

6. Exemple : mauvaise pratique (à éviter)
❌ “Ce fichier sert à expliquer un peu comment ça marche, mais c’est pas trop important.”  
→ Trop vague, ton non professionnel, manque de précision.

---

7. Exemple : bonne pratique (à suivre)
✔️ “Ce fichier décrit les étapes nécessaires pour initialiser le module. Il doit être lu avant toute modification du code source.”

---

8. Exemple : structure complète d’un fichier
`markdown

Titre du fichier

Objectif
Décrire clairement ce que contient le fichier.

Contexte
Expliquer pourquoi ce fichier existe.

Instructions
1. Étape 1
2. Étape 2
3. Étape 3

Notes
Informations complémentaires si nécessaire.
`
`
