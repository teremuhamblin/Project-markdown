1. Qu’est‑ce que Markdown ?

Markdown est un langage de balisage léger conçu pour écrire du texte simple, lisible, et facile à convertir en HTML, PDF ou documentation technique.

Objectifs :
- écrire rapidement
- rester lisible même sans rendu
- produire des documents propres et structurés
- standardiser la documentation

---

2. Pourquoi utiliser Markdown ?

Markdown est utilisé dans :
- GitHub (README, issues, PR)
- Documentation technique
- Notes de projet
- Wikis
- Blogs statiques
- Templates de documentation

Avantages :
- syntaxe simple
- compatible partout
- versionnable (Git)
- durable dans le temps

4. Organisation d’un document Markdown

Un document clair suit généralement cette structure :

1. Titre principal (#)
2. Introduction courte
3. Sections (##)
4. Sous‑sections (###)
5. Listes, exemples, code
6. Conclusion ou résumé

---

5. Bonnes pratiques

- écrire simple et clair  
- éviter les phrases trop longues  
- utiliser des titres cohérents  
- préférer les listes pour structurer  
- ajouter des exemples concrets  
- utiliser des chemins relatifs pour les images  
- mettre à jour régulièrement la documentation
