Modules Complets — Workflow d’Écriture • Guide Formateur • Normes Internes

Ce document regroupe trois modules essentiels pour structurer, enseigner et maintenir une documentation Markdown professionnelle.

---

MODULE 1 — Workflow d’Écriture

Ce module décrit le processus complet pour créer, structurer, valider et publier un document Markdown dans un projet professionnel.

---

1. Objectifs du workflow
- Standardiser la création de documents
- Assurer la cohérence entre les fichiers
- Faciliter la relecture et la maintenance
- Garantir une documentation durable et professionnelle

---

2. Étapes du workflow

2.1 Préparation
1. Identifier le sujet du document  
2. Vérifier s’il existe déjà un fichier similaire  
3. Choisir le bon emplacement dans l’arborescence  
4. Lire les normes internes et le style guide

---

2.2 Création du fichier
1. Dupliquer TEMPLATE.md  
2. Renommer le fichier selon les conventions  
3. Ajouter un titre principal (#)  
4. Ajouter une introduction courte

---

2.3 Structuration
- Utiliser une hiérarchie logique :
`

Titre

Section

Sous-section
`
- Ajouter un sommaire si le document dépasse 30 lignes  
- Aérer avec des listes, tableaux, citations, blocs de code  

---

2.4 Contenu
- Ajouter des exemples concrets  
- Ajouter des liens internes  
- Ajouter des images si nécessaire  
- Ajouter des blocs de code annotés  

---

2.5 Validation
Checklist :
- [ ] Titre principal unique  
- [ ] Structure cohérente  
- [ ] Liens internes fonctionnels  
- [ ] Orthographe vérifiée  
- [ ] Style conforme au STYLEGUIDE  
- [ ] Sections courtes et lisibles  

---

2.6 Publication
1. Commit du fichier  
2. Mise à jour du CHANGELOG.md  
3. Vérification du rendu GitHub  
4. Ajout dans l’index si nécessaire  

---

3. Bonnes pratiques
- Un fichier = un sujet  
- Sections courtes  
- Exemples concrets  
- Liens internes systématiques  
- Mise à jour régulière  

---

MODULE 2 — Guide Formateur

Ce module est destiné aux formateurs, enseignants ou responsables documentation qui doivent transmettre les bonnes pratiques Markdown.

---

1. Objectifs pédagogiques
- Comprendre les niveaux de difficulté (bases → intermédiaire → avancé)  
- Savoir expliquer clairement les concepts  
- Guider les apprenants dans la pratique  
- Évaluer les productions Markdown  

---

2. Structure d’un cours Markdown

2.1 Introduction
- Qu’est-ce que Markdown  
- Pourquoi l’utiliser  
- Où il est utilisé (GitHub, docs, notes, CI/CD…)

2.2 Démonstration
- Montrer un fichier brut  
- Montrer le rendu  
- Expliquer la logique “texte → structure”

2.3 Pratique guidée
- Exercices simples  
- Correction en direct  
- Comparaison avant/après  

2.4 Mise en situation
- Documentation réelle  
- README complet  
- Workflow GitHub  

---

3. Méthodes pédagogiques

3.1 Progression par niveaux
- Bases : syntaxe  
- Intermédiaire : tableaux, citations, code  
- Avancé : docs, GitHub, CI/CD  

3.2 Apprentissage par la pratique
- 70 % exercices  
- 30 % théorie  

3.3 Feedback structuré
- Lisibilité  
- Cohérence  
- Structure  
- Navigation  

---

4. Évaluation

4.1 Critères
- Structure correcte  
- Syntaxe maîtrisée  
- Liens internes fonctionnels  
- Exemples pertinents  
- Mise en page propre  

4.2 Barème
- 40 % structure  
- 30 % contenu  
- 20 % lisibilité  
- 10 % cohérence  

---

5. Ressources formateur
- TEMPLATE.md  
- STYLEGUIDE.md  
- Exercices (niveaux 1, 2, 3)  
- Projets pratiques  
- Module Markdown Pro  

---

MODULE 3 — Normes Internes

Ce module définit les standards internes pour garantir une documentation cohérente, lisible et maintenable dans tout le projet.

---

1. Conventions de nommage

1.1 Fichiers
- minuscules  
- tirets (-)  
- pas d’espaces  
- noms explicites  

Exemples :
`
installation.md
guide-utilisation.md
architecture-systeme.md
`

---

1.2 Dossiers
- un dossier = une catégorie  
- toujours un README.md dans chaque dossier  

---

2. Structure obligatoire d’un fichier

Chaque fichier doit contenir :

1. Titre principal (#)  
2. Introduction courte  
3. Sections structurées  
4. Exemples (code, tableaux, images)  
5. Conclusion ou résumé  

---

3. Style rédactionnel

3.1 Ton
- professionnel  
- clair  
- direct  
- sans jargon inutile  

3.2 Paragraphes
- courts  
- une idée par paragraphe  

3.3 Listes
- préférer les listes pour structurer  
- éviter les blocs de texte massifs  

---

4. Liens et navigation

4.1 Liens internes
Toujours utiliser des chemins relatifs.

4.2 Sommaires
Obligatoires pour les documents > 30 lignes.

4.3 Index global
Mettre à jour INDEX.md si un nouveau module est ajouté.

---

5. Images et ressources

5.1 Emplacement
Toutes les images doivent être dans :
`
assets/images/
`

5.2 Texte alternatif
Obligatoire pour l’accessibilité.

---

6. Maintenance

6.1 Changelog
Toute modification doit être documentée dans CHANGELOG.md.

6.2 Nettoyage
Supprimer les fichiers obsolètes.

6.3 Cohérence
Vérifier régulièrement :
- titres  
- liens  
- style  
- structure  

---

Conclusion

Ces trois modules — workflow d’écriture, guide formateur et normes internes — constituent la base d’un système documentaire professionnel, cohérent et scalable.
