# Projet Pratique 1 — Créer une Documentation Complète

## 🎯 Objectif
Créer une documentation technique complète en Markdown pour un projet fictif.

## 📌 Cahier des charges
La documentation doit contenir :
- un `README.md` clair
- un guide d’installation
- un guide d’utilisation
- une architecture
- un changelog
- un index interne

## 🧩 Étapes
1. Créer un dossier `docs/`
2. Ajouter :
   - `installation.md`
   - `utilisation.md`
   - `architecture.md`
   - `index.md`
3. Ajouter des liens internes entre les fichiers
4. Ajouter un sommaire dans chaque fichier
5. Ajouter au moins :
   - un tableau
   - un bloc de code
   - une image
   - une citation

## 📝 Rendu attendu
Un dossier `docs/` complet, cohérent, navigable.
