# Projet Pratique 2 — Markdown et GitHub

## 🎯 Objectif
Utiliser Markdown dans un environnement GitHub réel.

## 📌 Cahier des charges
Créer :
- un `README.md` professionnel
- un template d’issue
- un template de pull request
- un `CONTRIBUTING.md`
- un `SECURITY.md`

## 🧩 Étapes
1. Créer un dossier `.github/ISSUE_TEMPLATE/`
2. Ajouter :
   - `bug_report.md`
   - `feature_request.md`
3. Créer un fichier `.github/pull_request_template.md`
4. Ajouter des cases à cocher, des sections, des instructions
5. Ajouter des badges dans le README

## 📝 Rendu attendu
Un dépôt GitHub prêt pour la collaboration.
