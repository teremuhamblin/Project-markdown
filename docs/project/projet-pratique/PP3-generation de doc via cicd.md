# Projet Pratique 3 — Génération de Documentation via CI/CD

## 🎯 Objectif
Automatiser la génération ou la mise à jour de fichiers Markdown via CI/CD.

## 📌 Cahier des charges
Créer un workflow GitHub Actions qui :
- génère un fichier Markdown automatiquement
- met à jour un index
- commit les changements

## 🧩 Étapes
1. Créer `.github/workflows/docs.yml`
2. Ajouter un job :
   - qui exécute un script
   - qui génère `docs/auto.md`
3. Ajouter un script (bash ou python)
4. Commit automatique via GitHub Actions

## 📝 Rendu attendu
Un workflow fonctionnel qui génère du Markdown automatiquement.
