# USAGE

Ce document explique comment utiliser ce template **Project‑markdown** pour créer, organiser et maintenir une documentation claire et professionnelle.

## 1. Cloner ou utiliser comme template
- Cliquez sur **Use this template**
- Créez un nouveau dépôt basé sur cette structure
- Clonez-le localement si nécessaire

## 2. Structure recommandée
Le projet est organisé en plusieurs dossiers :
- `docs/` — documentation technique
- `project/` — structure interne et workflows
- `writing/` — règles d’écriture et exemples
- `assets/` — images, modèles, ressources

## 3. Fichiers essentiels
- `README.md` — présentation du projet
- `PLAN.md` — plan global
- `STYLEGUIDE.md` — conventions d’écriture
- `TEMPLATE.md` — modèle de document
- `CONTRIBUTING.md` — règles de contribution
- `SECURITY.md` — politique de sécurité

## 4. Comment créer un nouveau document
1. Copiez `TEMPLATE.md`
2. Renommez-le selon votre besoin
3. Remplissez les sections
4. Ajoutez-le dans le dossier approprié

## 5. Bonnes pratiques
- Garder les fichiers courts et structurés
- Utiliser des titres clairs
- Ajouter des liens internes
- Mettre à jour le `CHANGELOG.md` à chaque modification
