# Security Policy

Merci de contribuer à la sécurité de ce projet.

## 1. Signalement d’une vulnérabilité
Si vous découvrez une faille :
- Ne pas publier l’information publiquement
- Contacter directement le mainteneur
- Fournir une description claire et reproductible

## 2. Vulnérabilités concernées
- Exposition de données sensibles
- Scripts malveillants
- Mauvaise configuration
- Accès non autorisé

## 3. Ce qui n’est pas considéré comme une vulnérabilité
- Erreurs visuelles
- Problèmes mineurs non liés à la sécurité
- Bugs sans impact sur l’intégrité du projet

## 4. Processus de correction
1. Analyse du rapport
2. Reproduction du problème
3. Correctif en branche dédiée
4. Mise à jour du changelog
5. Publication d’un patch

Merci de contribuer à un projet plus sûr.
